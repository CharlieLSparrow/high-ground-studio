# Quipsly / QuipLore Visual Assets

Date organized: 2026-05-25

These files are reference assets for visual-language study. They are not
automatically production-safe illustrations, verified quote records, or final
brand assets.

## Asset Inventory

- `quipsly-character-states-brainstorm.png`
  - Character states, emotion set, app icons, notification, loading, empty
    state, nest, and basic palette.
- `quipsly-character-variety-system.png`
  - Character variation system: tones, markings, accessories, roles, cultural
    styling references, and silhouette guide.
- `quipsly-character-model-sheet.png`
  - Character model sheet with front, profile, back, and top-leaning views.
- `quipsly-quotable-people-grid-reference.png`
  - Grid of quotable-person-inspired Quipsly variants and quote cards.
- `quipsly-person-cosplay-card-grid.png`
  - Person-inspired Quipsly card grid for authors, thinkers, artists, and
    leaders.
- `quipsly-movie-tv-cosplay-card-grid.png`
  - Movie/TV-inspired Quipsly card grid. Treat as IP-sensitive reference only.
- `ip-reference-star-trek-card-grid.png`
  - Fandom-specific card grid. Treat as IP-sensitive reference only.
- `ip-reference-lotr-lesson-page.png`
  - Fandom-specific lesson/page composition. Treat as IP-sensitive reference
    only.
- `quipsly-home-introduction-page.png`
  - Quipsly world introduction page with hero, feature card, quote cards, and
    footer CTA.
- `quiplore-person-page-einstein-layout.png`
  - QuipLore quotable-person page layout prototype.
- `quiplore-quipsly-two-site-homepage-brainstorm.png`
  - Split QuipLore/Quipsly homepage concept and cross-site mission layout.
- `quipsly-auto-trace-full.svg`
  - Auto-traced vector experiment from character art; useful for studying
    shapes, not as final production SVG.
- `quipsly-auto-trace-simple.svg`
  - Simplified vector trace with an extractable starter palette.

## Notes

- Use these for product and visual direction.
- Do not treat quote text inside the images as verified quote data.
- Do not ship fandom/IP-specific images or exact costume likenesses without
  separate legal/editorial review.
- Production assets should eventually move under `apps/quiplore/public/brand`
  or `apps/quiplore/public/illustrations` once an app exists and the asset is
  approved for runtime use.
