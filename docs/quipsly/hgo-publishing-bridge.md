# Quipsly -> High Ground Odyssey publishing bridge

Quipsly Nest publishes public High Ground Odyssey episode pages through a public packet boundary.

## Rule

Private manuscript state, operator notes, assistant ledger details, and production scratch data do not go directly to HighGroundOdyssey.com. The public site receives `hgo-public-episode-packet-v1` records only.

## Current starter episode path

- Quipsly Publisher Mode calls `POST /api/hgo/publish-starter-episodes`.
- The endpoint upserts `HgoStagedProjectionArtifact` and `HgoEpisodePublishCandidate` records as `published`.
- HighGroundOdyssey.com reads published DB packets first, then falls back to checked-in packet JSON under `apps/web/content/publish/hgo-episodes`.
- The checked-in packets make Episodes 1-3 deploy-safe even if the DB bridge is temporarily unavailable.
- The starter packets are currently generated by `scripts/hgo-seed-public-episodes.mjs` from markdown files in `apps/web/content/_inbox`.

## Human editing truth

Today, Episodes 1-3 are not yet generated directly from the `/create` editor's tagged Chapter/Episode boundaries. The public pages are rendered from `hgo-public-episode-packet-v1` packets. To change the live content right now, edit the source markdown/packet path, regenerate the packets, publish through Nest, and redeploy HGO if the checked-in fallback needs to change.

The next product step is to let Publisher Mode preview a packet from the active manuscript boundary, show a diff, require operator approval, then write the same packet shape to `HgoEpisodePublishCandidate`.

## Public starter pages

- `/episodes/episode-1-write-it-down`
- `/episodes/episode-2-look-for-lessons`
- `/episodes/episode-3-chub-and-jack`

Each packet includes a YouTube ID, show-note beats, voice cards, public summary text, and the manuscript text covered by that episode.
