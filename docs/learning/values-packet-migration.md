# Values Packet Migration

This pass is a packet-first migration.

What that means in this repo:

- the canonical creative source now lives at `apps/web/content/episodes/learning-to-lead/values/packet.mdx`
- staging and inbox remain preserved upstream evidence
- `publish` was not touched, because packet creation and public derivative creation are two different acts

Why publish stayed untouched:

- there is no settled public title yet
- Charlie’s upstream material exists in two real shapes
- the research dossier is rich, but still production support rather than public copy

What “canonical creative source” means here:

- future work on the episode should start in the packet
- staging and inbox remain reference material, not the main editing surface
- a later public sync should selectively derive from the packet, not from raw staging files

What ambiguity was preserved on purpose:

- `Values` vs `The Values Toolkit`
- sidebars vs single-response Charlie shape
- which Scott value cluster should lead any future public page

Why this is a useful interview story:

- it shows migration discipline instead of accidental publishing
- it shows how to consolidate messy creative sources without deleting drafts
- it shows how to separate source-of-truth creation from presentation-layer release
